# 🧬 Hermes Agent 进化之路

> **从 Discord 草根社区到 20 万星开源顶流，46 天从 v0.1.0 冲到 v0.8.0，一个"会陪你一起长大"的 AI 智能体。**

---

## 一、前世：Nous Research 的基因 (2022-2025)

Hermes Agent 不是凭空诞生的，它背后是 **Nous Research** 长达三年的积累。

### 1.1 草根起源 (2022)

2022 年，一群 AI 研究者和工程师在 Discord、GitHub、Twitter 上自发聚集，基于 Meta 的 LLaMA 系列模型进行实验和微调。这不是典型的硅谷创业——这是一场去中心化的 AI 开源运动。

### 1.2 公司成立与创始团队 (2023)

2023 年正式注册公司，四位联合创始人背景独特：

| 创始人 | 角色 | 背景 |
|--------|------|------|
| **Jeffrey Quesnelle** | CEO | 前区块链基础设施工程师 |
| **Karan Malhotra** | Head of Behavior | 前 Stanford Brain Stimulation Lab |
| **Teknium** | Head of Post-Training | 前 Stability AI，OpenHermes 模型创造者 |
| **Shivani Mitra** | 联合创始人 | 核心团队成员 |

### 1.3 Hermes 模型系列：开源 AI 的里程碑

在 Agent 之前，Nous 以 **Hermes 模型系列** 闻名，Hugging Face 累计 **3300 万+ 次下载**：

| 版本 | 亮点 |
|------|------|
| **Nous-Hermes-13b** | 首个爆款，30 万+条指令微调 |
| **Hermes 2 Vision** | 轻量级视觉语言模型 |
| **Hermes 3** | 基于 Llama 3.1 (8B/70B/405B)，合成数据微调 |
| **Hermes 4** | 混合推理模型，50 倍训练数据 |
| **Hermes 4.3-36B** | **首个完全在 Psyche 去中心化网络上训练的模型** |

### 1.4 Psyche：去中心化训练的野心

Psyche 是 Nous 最具野心的基础设施项目——基于 **Solana 区块链** 的去中心化 AI 训练网络：

- **DisTrO 优化器**：将 GPU 间通信数据压缩 1,000-10,000 倍
- **双主网络架构**：Solana 智能合约管理共识 + P2P mesh 传输梯度
- **实际表现**：24 节点训练 Hermes 4.3，性能超越传统集中式训练

### 1.5 融资与估值 (2025)

2025 年 4 月，Nous Research 完成 **Paradigm** 领投的 **$50M Series A**，估值 **$10 亿**。团队约 20 人。

## 二、诞生：v0.1.0 首次开源 (2026.2.25)

### 2.1 发布背景

2026 年 2 月 25 日，Nous Research 正式将 **Hermes Agent v0.1.0** 以 MIT 许可在 GitHub 开源。仓库早在 2025 年 7 月 22 日创建但长期私有。

### 2.2 核心设计哲学

> "The agent that grows with you."

Hermes Agent 从第一天就定下了与所有竞品不同的核心哲学：

- **不是聊天机器人**——是一个持续运行的、私有化部署的数字同事
- **用完不忘**——跨会话持久记忆，FTS5 全文搜索 + LLM 摘要
- **越用越强**——完成复杂任务后自动生成技能，技能在使用中自我改进
- **不属于某一台机器**——属于你这个人，多平台统一身份

### 2.3 最初的武器库

- 基础 Agent 框架：`run_agent.py` 核心循环
- CLI 终端界面：`cli.py`
- 核心工具：终端执行、文件操作、网页搜索
- SQLite + FTS5 会话存储
- 多 LLM 提供商支持

v0.8.0 之后，版本号发生了跳跃（v0.8.0 → v0.10.0 → ... → v0.15.0），这段时间是 Hermes 从"Agent 框架"进化为"Agent 平台"的关键期。

### 4.1 关键事件

| 时间 | 事件 | 影响 |
|------|------|------|
| 4.15 | **微信原生支持** | 微信扫码链接，私聊群聊全支持，图片/视频/文件/语音全覆盖 |
| 4 月中 | **EvoMap 争议** | 中国团队 EvoMap 指控代码结构雷同，Nous 否认，社区热议 |
| 4 月下 | **OpenRouter 日消耗第二** | Hermes Agent 的 token 消耗量在 OpenRouter 日榜排第二 |
| 4 月 | **黑客马拉松** | Nous Research 举办首届 Hermes Agent 黑客松 |

### 4.2 GEPA 自我进化引擎 🔬

在 v0.8.0 之后引入的核心能力——**GEPA (Genetic-Pareto Prompt Evolution)**：

```
读取当前技能/prompt/tool ──→ 生成评估数据集
                                      │
                                      ▼
                                 GEPA 优化器 ◄── 执行轨迹
                                      │              ▲
                                      ▼              │
                                 候选变体 ──→ 评估
                                      │
                                 约束门 (测试/大小/基准)
                                      │
                                      ▼
                                 最佳变体 ──→ 自动 PR
```

- 由 UC Berkeley、Stanford、MIT 研究者联合开发
- **ICLR 2026 Oral 论文**
- 无需 GPU 训练，纯 API 调用，**$2-10/次优化**
- 仅需 100-500 次评估即可收敛（传统 RL 需上万次）

### 4.3 技能系统成熟

技能从"自动保存"进化为完整的生命周期管理：

1. **自动创建**：复杂任务完成后自动提炼
2. **使用中改进**：发现更优路径自动 patch
3. **社区共享**：遵循 agentskills.io 开放标准
4. **Curator 守护**：后台自动维护（标记过时、归档、备份）

### 4.4 记忆系统四层架构

| 层级 | 名称 | 内容 | 特点 |
|------|------|------|------|
| 第一层 | 常驻提示记忆 | MEMORY.md + USER.md | 限制 3575 字符，强制筛选重要信息 |
| 第二层 | 会话归档 | SQLite + FTS5 全文索引 | LLM 摘要，跨会话检索 |
| 第三层 | 技能库 | SKILL.md 文件集合 | 程序性记忆，任务匹配自动调用 |
| 第四层 | 用户画像 | Honcho 辩证用户建模 | 跨会话自动加深对用户的理解 |

## 八、争议与反思

### 8.1 EvoMap 争议 (2026.4.15)

中国 AI 团队 EvoMap 发布长文，指控 Hermes Agent 与其开源项目 Evolver 在架构上"结构性同构"：

- **争议焦点**：闭环学习范式、三层记忆体系、技能自我改进机制
- **Nous 回应**：联合创始人 Teknium 否认，称"从未听说过这个项目"
- **技术事实**：Hermes 仓库 2025.7 创建但长期私有，争议功能 3 月中旬才完整推出

这个争议反映了开源 AI Agent 领域"趋同进化"的困境——当大家都朝着"持久记忆+技能学习"这个正确方向走，架构自然会相似。

### 8.2 教训与启示

1. **速度就是护城河**：46 天 8 个大版本，社区驱动的高速迭代本身就是壁垒
2. **开源社区的乘数效应**：200+ 社区贡献者，单周 300+ PR，远非闭源团队可比
3. **模型+Agent+基础设施三合一**：从 Hermes 模型到 Psyche 网络到 Agent，形成独特生态闭环
4. **"用完不忘"是正确方向**：可持续学习型智能体比更好的对话体验更有长远价值
